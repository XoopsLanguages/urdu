تفصیل

پروفائل ماڈیول حسب ضرورت صارف پروفائل فیلڈز کے انتظام کے لیے ہے۔

انسٹال/اَن انسٹال کریں۔

کسی خاص اقدامات کی ضرورت نہیں، معیاری تنصیب کے عمل پر عمل کریں - /profile فولڈر کو ../modules ڈائریکٹری میں نکالیں۔ ایڈمن -> سسٹم ماڈیول -> ماڈیول کے ذریعے ماڈیول انسٹال کریں۔

ماڈیولز انسٹال کرنے کے بارے میں تفصیلی ہدایات میں دستیاب ہیں۔ XOOPS آپریشنز مینوئل

آپریٹنگ ہدایات

اس ماڈیول کو ترتیب دینے کے لیے آپ کو:

  • ماڈیول کے لیے اپنی ترجیحات کو ترتیب دیں (دیکھیں 'ترجیحات') اور اختیاری طور پر پروفائل بلاک اگر آپ اسے استعمال کرنا چاہتے ہیں (دیکھیں 'بلاک')۔
  • موجودہ زمروں میں ترمیم کریں یا نئے شامل کریں۔
  • موجودہ فیلڈز میں ترمیم کریں یا نئے شامل کریں۔ یہاں آپ یہ بتا سکتے ہیں کہ کون سے فیلڈز کس زمرے میں نظر آئیں گے، اور اگر وہ صارف کے رجسٹریشن کے دوران نظر آئیں گے۔
  • رجسٹریشن کے مراحل کی ترتیب کی وضاحت کریں۔
  • اور آخر میں، آپ انفرادی فیلڈز کے لیے اجازتیں ترتیب دے سکتے ہیں - کون سے قابل تدوین ہیں، کون سے قابل تلاش ہیں۔

سپیم مخالف اقدامات

سپیم رجسٹریشن کو کم کرنے کے لیے، درج ذیل کام کریں:

  • ایڈمن میں پروٹیکٹر ماڈیول پر جائیں، ترجیحات پر جائیں، اور پھر نیچے، اس آپشن پر: "فورم سپیم کو روکیں" www.stopforumspam.com ڈیٹا بیس پر رجسٹرڈ اسپامرز کے خلاف POST ڈیٹا چیک کرتا ہے۔
    اسے "آئی پی پر پابندی (کوئی حد نہیں)" پر سیٹ کریں

  • /class/captcha/config.php میں، یقینی بنائیں کہ موڈ "ٹیکسٹ" کے طور پر سیٹ ہے:

    واپسی $config = array(
    'disabled' => غلط، // کیپچا کو غیر فعال کریں۔
    'موڈ' => 'ٹیکسٹ'، // ڈیفالٹ موڈ، آپ 'ٹیکسٹ'، 'امیج'، 'ری کیپچا' کا انتخاب کرسکتے ہیں (api کلید کی ضرورت ہے)
    'name' => 'xoopscaptcha'، // captcha کا نام
    'skipmember' => سچ ہے، // اراکین کے لیے کیپچا چیک کو چھوڑیں
    'maxattempts' => 10، // ہر سیشن کے لیے زیادہ سے زیادہ کوششیں
    );

  • پروفائل ماڈیول میں، ایڈمن پر جائیں اور بنیادی قدم میں جو یہاں موجود ہے:
    /modules/profile/admin/step.php?id=1
    "اسٹاپ کے بعد محفوظ کریں" کو "نہیں" پر سیٹ کریں


  • پروفائل کی ترجیحات میں، "دوسرے رجسٹریشن مرحلے کے بعد کیپچا استعمال کریں" کے اختیار کو "ہاں" پر سیٹ کریں (یہ پہلے سے طے شدہ ہے)
